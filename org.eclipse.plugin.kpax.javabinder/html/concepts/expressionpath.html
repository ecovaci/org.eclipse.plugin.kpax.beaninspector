JavaBean Path Expression

The JavaBean

It is known that a JavaBean is a Java class following the conventions:

  1. The class must have a public default constructor (with no arguments).
  2. The class properties must be accessible using get, set, is (can be used for boolean properties instead of get), and other methods (so-called accessor methods and mutator methods) according to a standard naming convention. Setters can have one or more than one argument.
  3. The class should be serializable.

The Path Expression

For a JavaBean, a Path Expression is a property navigation path. Such path is represented by a String made of dot-separated names of properties of the Java Bean class.
For example, let's take the following JavaBeans:

class Country implements Serializable {
	private String code;
	private String label;
	
	//-- Accessor methods here
}

class Address implements Serializable {
	private Country country;
	private String town;
	
	//-- Accessor methods here
}

class Person implements Serializable {
	private String firstName;
	private String lastName;
	private Address address;
	
	//-- Accessor methods here
}

Some path expressions, considering the Person class as root, are:

  • firstName
  • address
  • address.town
  • address.country.code

For the sake of simplicity, we'll refer path expressions as paths.

The JavaBean revisited

Let us look back to the JavaBean definition from the path expressions perspective.
It's obvious that the 1th and the 3th requirements have nothing to do with paths. That's why will keep only the 2th requirement.
Therefore, the JavaBean definition this plugin will take it is: any Java class or interface having at least one property.
